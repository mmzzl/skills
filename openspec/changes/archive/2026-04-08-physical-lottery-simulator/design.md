## Context

本项目从零开始构建一个基于3D物理引擎的摇号模拟器。不依赖现有物理引擎库（如 PyBullet、Cannon.js），使用纯 numpy 实现向量运算和碰撞检测。目标是生成一个可复用的 Python 模块，支持前区/后区多组独立摇号。

## Goals / Non-Goals

**Goals:**
- 实现准确的3D球体碰撞物理模拟（球-球、球-容器壁）
- 模拟真实气流场，使球在球形容器内翻滚，仅中心区域球能被吹出
- 支持可配置的号码范围（如1-35选5、1-12选2）
- 纯数值运行，无需可视化，输出最终选中的号码
- 代码可作为 skill 的核心引擎被调用

**Non-Goals:**
- 不提供可视化渲染（2D/3D渲染均不在范围内）
- 不实现GPU加速（CPU足够处理47个球的模拟）
- 不模拟空气阻力、摩擦力等次要物理效应
- 不处理多个容器同时运行（单次只跑一组摇号）

## Decisions

### 1. 纯 numpy 实现，不引入物理引擎库

选择 numpy 而非 PyBullet/MuJoCo，原因：
- 47个球的模拟量极小，不需要专业物理引擎
- 减少外部依赖，模块更轻量
- 完全控制物理参数，便于调优出"摇号感"

### 2. 3D向量用 numpy 数组而非自定义 Vec3 类

```python
# 方案A: numpy数组（选择）
pos = np.array([0.0, 0.5, 0.0])
vel = np.array([0.1, 0.2, 0.0])

# 方案B: 自定义Vec3类
pos = Vec3(0.0, 0.5, 0.0)
```

选择 numpy 数组：性能更好，代码更简洁，且 numpy 已经是依赖。

### 3. 碰撞检测用 O(N²) 暴力法，不用空间分割

47个球最多 1100 对检测，在现代CPU上每帧 < 1ms。引入四叉树/八叉树反而增加复杂度。

### 4. 气流场用高斯分布 + 高度因子

```
force = exp(-d² / (2σ²)) × height_factor × power
```

- σ 控制气流集中度（默认 0.3 × 容器半径）
- height_factor 随球高度增加（底部弱、顶部强）
- 添加微小随机扰动使球翻滚

### 5. 固定时间步长 + 子步进

- 主时间步长 dt = 1/60s
- 每帧 4 个子步进（substep），提高碰撞精度
- 避免高速球穿透容器壁

### 6. 出球后立即补充新球（可选） vs 不补充

选择不补充。前区35个球选5个，后区12个球选2个，球数足够，不需要补充机制。

### 7. 时序约束与自动调优

硬性要求：每 10 秒必须出一个球，20 秒不出球视为故障。

```
时间线:
0s ─────────── 10s ─────────── 20s
     正常出球      自动增压        故障重启
                  (+20%气流)

┌─────────────────────────────────────────────┐
│  Timer 启动                                  │
│     │                                        │
│     ▼                                        │
│  10s 内出球? ──YES──▶ 重置 Timer，继续        │
│     │                                        │
│     NO                                       │
│     ▼                                        │
│  气流 power *= 1.2                           │
│     │                                        │
│     ▼                                        │
│  再等 10s (累计20s)                           │
│     │                                        │
│     ▼                                        │
│  出球了? ──YES──▶ 继续                        │
│     │                                        │
│     NO                                       │
│     ▼                                        │
│  故障 → 新 seed 重启 (最多重试3次)             │
└─────────────────────────────────────────────┘
```

## Risks / Trade-offs

| Risk | Impact | Mitigation |
|------|--------|------------|
| 气流参数不当导致球全飞出去或不飞 | 模拟失效 | 自动增压机制 + 参数化默认值 |
| 球重叠/穿透（高速碰撞） | 物理不准确 | 子步进 + 重叠分离修正 |
| 10秒内球出不来 | 违反时序约束 | 10s自动增压20%，20s故障重启 |
| 连续3次重启仍失败 | 无法完成摇号 | 抛出UnrecoverableError，携带诊断信息 |
| numpy 在 Windows 上的性能 | 轻微影响 | 47个球影响可忽略 |
