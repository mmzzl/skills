# FMEA 审核 Checklist

- [ ] 是否覆盖所有关键功能点/业务？
- [ ] 功能要求是否可验证，是否包含成功判定？
- [ ] 失效模式是否描述为“不希望出现的现象”？
- [ ] 失效影响是否写到客户业务影响，而不是只写技术现象？
- [ ] 失效原因是否一行一个、可定位、可改进？
- [ ] S/O/D 是否均为 1~10 整数？
- [ ] S/O/D 评分是否能在 S/O/D 评价准则中找到依据？
- [ ] AP 是否由 AP 矩阵查表得到？
- [ ] AP=H 是否有明确改进措施？
- [ ] AP=M 是否有改进措施或风险接受说明？
- [ ] S=9~10 且 AP=H/M 是否提示管理层评审？
- [ ] 是否补充开发自检结论？
- [ ] 是否补充测试用例 ID 或用例覆盖说明？
